# Chapter 01 Narration

## Beat 1

[T=00:00] A generalist robot is useful when the same task intent survives a new scene, a new object layout, or a different body.

## Beat 2

[T=00:12] Direct replay stores a path. Learning-from-Observation treats that path as evidence for purpose, then abstracts away from the human trajectory.

## Beat 3

[T=00:25] The encoder compresses language and vision into a compact task model. The model must be small enough to move across environments and robots.

## Beat 4

[T=00:39] The first field is `what-to-do`: the primitive operation the task requires, such as open, pick, place, bring, or insert.

## Beat 5

[T=00:51] The second field is `where-to-do`: the grounded object, surface, handle, axis, or target region that makes the operation executable.

## Beat 6

[T=01:03] The skill-agent library supplies `how-to-do`. Each skill decodes a symbolic request into feedback-driven robot action.

## Beat 7

[T=01:18] Because the task model is not a motor program, the same request can produce different feasible motions on different robot bodies.

## Beat 8

[T=01:31] Chapter one ends with the interface: `M = (what, where)`. Chapter two asks how the primitive skill vocabulary should be defined.
