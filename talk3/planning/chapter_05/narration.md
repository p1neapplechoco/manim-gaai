# Chapter 05 Narration

## Beat 1

[T=00:00] Grasping is not one hand pose. It is a family of skills for different object and task conditions.

## Beat 2

[T=00:11] A taxonomy names grasp styles, but task purpose chooses which style matters.

## Beat 3

[T=00:22] Closure makes that purpose testable: the object must be constrained enough for the intended action.

## Beat 4

[T=00:33] The library prepares several grasp agents, each covering a distinct closure strategy.

## Beat 5

[T=00:45] Contact-web agents represent fingers and object regions as a graph, making grasp structure explicit.

## Beat 6

[T=00:57] Superquadrics compress object shape into parameters a grasp policy can use.

## Beat 7

[T=01:09] CNN perception estimates shape and grasp parameters; RL tunes the action policy.

## Beat 8

[T=01:21] Within a grasp-purpose family, the same agent can transfer across many shapes.

## Beat 9

[T=01:31] Force strategy becomes part of the interface. Chapter six plugs these agents into the full robot system.
