# V5 Transition Contracts

Each chapter ends with one visual object that must reappear at the start of the next chapter. This makes the series feel like one derivation rather than nine disconnected explainers.

| From | End-state object | Next chapter start-state | Continuity purpose |
|---|---|---|---|
| 01 -> 02 | Task model card: `M = (what, where)` plus skill-library slot | The card asks what counts as a primitive skill | Moves from architecture to the state-transition basis of the library |
| 02 -> 03 | Filtered contact graph: `49 -> 20 -> 6` | The graph passes through a semantic constraint gate | Adds tool/environment common sense beyond contact physics |
| 03 -> 04 | Constraint gate labeled `physical + semantic` | The gate outputs a trainable policy target | Moves from feasibility constraints to agent training |
| 04 -> 05 | Policy surface with feedback arrow | The surface becomes a grasp-agent interface | Shows grasping as another reusable skill family |
| 05 -> 06 | Grasp agent card with object-shape distribution | The card plugs into an errand robot system map | Moves from individual skill generalization to integrated robot behavior |
| 06 -> 07 | Errand system map: recognition -> task model -> skill agents -> humanoid | The map collapses into the question: what is task essence? | Moves from system integration to historical abstraction |
| 07 -> 08 | State transition `T: s_t -> s_{t+1}` | The symbol is placed inside multiple domain state spaces | Generalizes essence beyond manipulation |
| 08 -> 09 | Domain-state atlas | The atlas becomes the representation side of the final factorization | Moves from domain representations to transfer and synthesis |

## Opening Rule

The first 8-10 seconds of each chapter should visually acknowledge the previous handoff before introducing new content. The narration should say what the object meant and what question remains.

## Closing Rule

The final 10-15 seconds of each chapter should state:

1. What was established.
2. What is still missing.
3. Which object carries that question forward.
